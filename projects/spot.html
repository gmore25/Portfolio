Project

Spot Sensor Payload Integration

Built Python + ROS software to integrate voltage + vibration sensors as a payload on Boston Dynamics Spot for inspection workflows in safety-critical environments.

PythonROSSensors LinuxValidation
Slide 1

Problem

  • Needed reliable sensor readings during robot-assisted inspections.
  • Hardware + software integration required robustness (dropouts, noise, environmental effects).
  • Goal: support safer workflows by shifting inspection time away from high-exposure areas.
Slide 2

System architecture

  • ROS nodes handled acquisition, formatting, timestamping, and publishing data.
  • Downstream consumers could log, monitor, and validate readings during runs.
  • Vendor SDK access was coordinated by a senior engineer; I developed within that integration framework.
Slide 3

What I implemented

  • Python ROS node(s) for sensor ingestion and topic publishing.
  • Sanity checks + error handling (timeouts, missing data, unexpected ranges).
  • Logging hooks for repeatable troubleshooting and validation.
Slide 4

Testing & validation

  • Started with mocked inputs / replayable sample streams to validate parsing + formatting.
  • Incremental integration: component test → on-robot test → end-to-end run validation.
  • Used logs to isolate software vs hardware vs environment issues.
Slide 5

Outcome

  • Enabled sensor-backed inspection runs and improved confidence in collected readings.
  • Supported workflows that reduced time humans spent in high-exposure environments.
  • Next: automate calibration checks and add regression tests for sensor pipelines.
